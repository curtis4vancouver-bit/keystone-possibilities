Automated Video Reframing and Spatial Editing

Converting standard horizontal 16:9 documentary or podcast footage into mobile-native vertical 9:16 format is the foundational automation task. Simple center-cropping frequently ruins the composition by cutting off the primary subject or essential context. The system must employ sophisticated spatial reframing techniques.

The most reliable, computationally efficient aesthetic solution is the "BlurPad" technique. This method preserves the entire 16:9 frame by placing the uncropped video in the absolute center of the vertical canvas, subsequently filling the remaining empty space at the top and bottom with a heavily blurred, scaled-up version of the identical video feed.   

The exact FFmpeg filter_complex command, which the AI agent can execute via the Python subprocess module, is structured as follows:

Bash
ffmpeg -i input_16x9.mp4 -filter_complex "[0:v]scale=1080:1920:force_original_aspect_ratio=increase,boxblur=30:1,crop=1080:1920[b];[0:v]scale=1080:1920:force_original_aspect_ratio=decrease[fg];[b][fg]overlay=(W-w)/2:(H-h)/2" -c:v libx264 -crf 21 -preset fast -c:a copy output_9x16_blurpad.mp4


This specific command first takes the input stream [0:v], scales it to fill a 1080x1920 frame, applies a massive boxblur with a radius of 30, and crops it to establish the background layer [b]. It then takes the same input stream [0:v], scales it down to fit within the 1080 width bounds, creating the foreground layer [fg]. Finally, it overlays the clear foreground perfectly centered over the blurred background.

For more advanced, dynamic cropping requirements, the pipeline utilizes moviepy v2.0 integrated with the ultralytics YOLOv8 computer vision model to track the subject's Region of Interest (ROI). This allows the script to dynamically adjust the center point of the vertical crop on a frame-by-frame basis, ensuring the human subject never leaves the visible frame.

from ultralytics import YOLO
from moviepy.editor import VideoFileClip, concatenate_videoclips
from moviepy.video.fx.crop import crop

# Initialize the YOLOv8 AI object detection model and load horizontal source video
model = YOLO("yolov8n.pt")
clip = VideoFileClip("source_16x9.mp4")
tracked_clips =

w, h = clip.size
new_w = int(h * 9 / 16) # Calculate exact vertical width required
new_h = h

for frame_no, frame in enumerate(clip.iter_frames()):
    results = model(frame)
    if results.boxes:
        # Identify the primary object bounding box based on highest confidence score
        main_obj = max(results.boxes, key=lambda x: x.conf)
        x1, y1, x2, y2 = [int(val) for val in main_obj.xyxy.tolist()]
        
        # Calculate dynamic center point of the tracked subject
        x_center = x1 + ((x2 - x1) / 2)
        
        # Execute bounds checking to prevent the crop region from exceeding video dimensions
        if x_center + (new_w / 2) > w:
            x_center = w - (new_w / 2)
        elif x_center - (new_w / 2) < 0:
            x_center = new_w / 2
            
        start_time = frame_no / clip.fps
        end_time = (frame_no + 1) / clip.fps
        subclip = clip.subclip(start_time, end_time)
        
        # Apply spatial crop perfectly centered on the moving subject
        cropped_clip = crop(subclip, x_center=x_center, y_center=h/2, width=new_w, height=new_h)
        tracked_clips.append(cropped_clip)

# Render the final concatenated output video
final_vertical_clip = concatenate_videoclips(tracked_clips, method="compose")
final_vertical_clip.write_videofile("auto_tracked_short.mp4", codec="libx264", audio_codec="aac")
