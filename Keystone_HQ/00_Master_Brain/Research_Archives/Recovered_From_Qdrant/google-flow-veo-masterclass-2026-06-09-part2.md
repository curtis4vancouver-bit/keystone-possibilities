GOOGLE FLOW 2026 PROFESSIONAL WORKFLOW FOR 24-CLIP MUSIC VIDEOS

== INDUSTRY STANDARD: PARALLEL GENERATION + NLE ASSEMBLY ==
Professionals NEVER use long Extend chains for music videos. They generate independent clips in parallel and stitch in post-production.

Why:
1. Speed: 12 concurrent jobs vs 1 sequential chain
2. Isolated failures: broken clip doesn't kill downstream chain
3. Music videos need fast cuts, varied angles, jump cuts - not one-take
4. Post-production transitions (whip pans, glitch, light leaks) mask clip boundaries

== PHASE 1: PRE-PRODUCTION (FREE/CHEAP) ==
- Use Flow Agent (no video credits) to build 24-shot storyboard from audio tempo + lyrics
- Generate character/location/style images with Nano Banana Pro (cheap)
- Save as Ingredients: @LeadSinger, @CyberpunkClub, @DesertHighway etc.
- This locks the visual law for entire project

== PHASE 3: EXTERNAL POST-PRODUCTION (DaVinci Resolve) ==
- Download all 24 raw MP4s from Flow
- Import into DaVinci Resolve
- Hard cuts on audio downbeats
- Speed ramps synced to musical transients
- Cross-dissolves, film burns, light leaks, glitch impacts at cut points
- Only return to Flow for specific surreal morph transitions (First+Last frame)

ECHNIQUES (WITHOUT EXTEND) ==

4. Standardized Camera Prompts: consistent motion descriptors across all clips
5. Style Drawer: Same reference image for color grading consistency
6. First+Last Frame: Upload end frame of A and start frame of B → interpolated bridge

== FRAMES-TO-VIDEO TIPS ==
- Costs same as Text-to-Video (15-30 credits Omni Flash)
- Great for specific morph transitions but expensive at scale (360-720 credits for all 24 clips)
- Use sparingly for thematic moments, NOT as default transition method
- Music videos should use hard cuts synced to beat, not interpolated transitions
